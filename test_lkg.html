1번 케이스

설명: 화면에서 알파벳 X가 보이는 버튼이며, aria-label이 '닫기'로 적용되어 있음

샘플:

<button aria-label="닫기">X</button>



2번 케이스

설명: '이메일' 텍스트 아래 입력란이 있음. 입력란에는 value 속성이 '연락처'로 적용되어 있고 추가로 aria-label이 '이메일'로 제공됨

샘플:

이메일

<div>이메일</div>
<input type="text" aria-label="이메일" value="연락처" />



3번 케이스

설명: W3C로고 이미지 링크가 있고, 이미지 alt 값은 'w3c 로고'이며 링크에는 aria-label이 'W3C 홈페이지'로 제공됨

샘플:

<a href="https://w3.org" aria-label="W3C 홈페이지">
<img src="w3c-logo.png" alt="w3c 로고" />
</a>



4번 케이스

설명: 화면에는 '사업자등록번호'라는 텍스트 레이블이 있고 그 아래에 3개의 빈 입력폼이 나란히 있음

샘플:

사업자 범용 공동인증서로 로그인하여 제안서를 접수하여 주십시오.

- -

<p>사업자 범용 공동인증서로 로그인하여 제안서를 접수하여 주십시오.</p>
<label for="co-num">사업자등록번호</label>
<div>
<input type="text" name="tpRgstNo1" maxlength="3" value="">
<span>-</span>
<input type="text" name="tpRgstNo2" maxlength="2" value="">
<span>-</span>
<input type="text" name="tpRgstNo3" maxlength="5" value="">
</div>



5번 케이스

설명: 다음 텍스트 링크에 aria-label이 '다음 페이지'로 적용됨

샘플:

다음

<a href="https://w3.org" aria-label="다음 페이지">다음</a>



6번 케이스

설명: 화면에는 '이메일 주소'라는 텍스트 레이블이 있고 그 옆에 이메일 아이디를 입력하는 첫 번째 입력란, @ 텍스트, 도메인을 입력하는 두번째 입력란, 이메일 도메인을 선택할 수 있는 셀렉트 박스가 순서대로 배치 됨. 첫번재 두번째 입력란에는 placeholder로 '입력해 주세요'가, 셀렉트 박스는 첫번째 옵션인 '직접 입력'이 화면에 보여짐

샘플:

@

<label for="emailPart">이메일 주소</label>
<input type="text" id="emailPart" placeholder="입력해 주세요.">
<span class="txt_at">@</span>
<input type="text" id="emailDomain" placeholder="입력해 주세요.">
<select title="이메일 도메인">
<option>직접 입력</option>
<option>gmail.com</option>
<option>kakao.com</option>
<option>naver.com</option>
</select>



7번 케이스

설명: 인스타그램 썸네일 이미지 목록 아래 SHOW MORE이라 표시된 이미지 버튼 있음. 버튼에aria-label="Load more Instagram posts"로 적용됨

샘플:

<button aria-label="Load more Instagram posts">
<img src="more.png" alt="">
</button>
